Running With Rifles Stats REST API

Documentation about the Running With Rifles Stats (RWRS) REST API. About RWRS

Access restriction ¶

Although this API is public, to prevent abuses an API token is required. Contact me in private (Discord is the preferred way) with a short message explaining what do you plan to create, the purpose of your project along its name.

You are required to mention somewhere in your project - in a visible manner - that you’re using the RWRS REST API.

Sending the API token ¶

All endpoints needs a valid API token, otherwise a 403 HTTP status code will be returned. Send this HTTP header in all of your HTTP requests along your API token:

Authorization: Token {token}

Endpoint ¶

The RWRS REST API endpoint is https://rwrstats.com/api.

Using HTTPS is mandatory or you’ll encounter redirections (which may not be appreciated by your HTTP client).

Rate limiting ¶

To prevent abuse, all resources are rate-limited. Each IP address + API token couple is limited to 1 request per second. Failing to respect this rate-limiting window will lead to a 429 HTTP status code.

Cache ¶

Some resources are subject to server-side caching (i.e you’ll not necessarily get the most up-to-date data from the server). Check their respective documentation for more information.

HTTP status codes ¶

The HTTP response status code must be used to check if the sent request was successfully handled or not:

  • 2xx Everything is OK. The response body will contain whatever the resource provide (check its doc)

  • 4xx The client made a mistake in its request, typically it’s an input parameter validation error

  • 5xx There were a server error

The RWRS REST API doesn’t provide internal error codes, but tries to make a meaningful use of HTTP status codes instead.

Input & output data format ¶

Everything is JSON along a Content-Type HTTP header of application/json.

Success (2xx) ¶

The output can be different regarding the resource. Check their respective doc.

Client error (4xx) ¶

Typically an input parameters validation error:

{
    "message": {
        "blow": "Missing required parameter in the JSON body",
        "hey": "Missing required parameter in the JSON body"
    }
}

In this case the hey and blow input parameters are missing.

If a query string parameter is missing or invalid, it’s the same output:

{
    "message": {
        "active": "Missing required parameter in the query string"
    }
}

May also happen in other cases like the requested resource wasn’t found:

{
    "message": "This thing does not exists."
}

Server error (5xx) ¶

They should not happen (exception for 503). If so, please get in touch.

Example:

{
    "message": "Error creating this thing: IndexError: out of bounds"
}

Servers ¶

A Server represent a public multiplayer Running With Rifles game.

Servers  ¶

Get the list of Servers
GET/servers{?location,map,type,mode,dedicated,ranked,not_empty,not_full,limit}

Return the list of Servers. The list may be optionally filtered by one or more criteria. The criteria is taken into account only if it is present in the query string.

Server-side caching ¶

This resource has a cache lifetime of 1 minute.

Example URI

GET https://rwrstats.com/api/servers?location=continent:eu&map=map10&type=vanilla&mode=COOP&dedicated=true&ranked=true&not_empty=true&not_full=true&limit=10
URI Parameters
HideShow
location
string (optional) Default: any Example: continent:eu

Servers country or continent of residence. This criteria format is either:

  • A concatenation of a location type and a location code, separated by a colon, OR

  • Just a country code OR

  • any

Available location types are:

  • continent

  • country

Available continent codes:

  • af - Africa

  • na - North America

  • oc - Oceania

  • an - Antarctica (who knows!)

  • as - Asia

  • eu - Europe

  • sa - South America

Country code must conform to ISO 3166-1 alpha-2.

map
string (optional) Default: any Example: map10

Servers map ID.

See also this file, part of the RWRS source code, which contains all the RWR maps metadata stored in the JSON format.

Choices: any lobby_2p island1 island2 island3 island4 island5 island6 island7 island8 pvp1 map1 map1_2 map2 map3 map4 map5 map6 map7 map8 map9 map10 map11 map12 map13 map14 map15 rwd_map1 def_dday def_lab viper def_lab_koth

type
string (optional) Default: any Example: vanilla

Servers type (used mod or DLC).

Choices: any vanilla vanilla.winter pvp pacific man_vs_world_mp Running_with_the_Dead overlord_defense lab_defense lab_def_koth viper

mode
string (optional) Default: any Example: COOP

Servers mode (game mode).

Choices: any COOP DOM PvP PvE PvPvE

dedicated
string, number (optional) Example: true

Whether to filter dedicated servers only. true or 1 to enable this filter. false or 0 will be ignored and thus disable this filter.

Choices: true 1 false 0

ranked
string, number (optional) Example: true

Whether to filter official servers only. true or 1 to enable this filter. false or 0 will be ignored and thus disable this filter.

Choices: true 1 false 0

not_empty
string, number (optional) Example: true

Whether to filter non-empty servers only. true or 1 to enable this filter. false or 0 will be ignored and thus disable this filter.

Choices: true 1 false 0

not_full
string, number (optional) Example: true

Whether to filter non-full servers only. true or 1 to enable this filter. false or 0 will be ignored and thus disable this filter.

Choices: true 1 false 0

limit
number (optional) Example: 10

Return the n first servers of the filtered list.

Players ¶

A Player represent an online multiplayer Running With Rifles account.

Important note

Players stats are updated daily and only concerns ranked (official) Invasion and Pacific servers.

Also, stats history is only recorded for the 5000 most experienced players.

Player  ¶

Get a specific Player
GET/players/{database}/{username}{?date}

Return the Player corresponding to the given database / username couple.

Server-side caching ¶

This resource has a cache lifetime of 1 hour.

Example URI

GET https://rwrstats.com/api/players/invasion/EPOCDOTFR?date=2018-11-30

string (required) Example: EPOCDOTFR

The player username.

date
string (optional) Example: 2018-11-30

ISO 8601 date for which to retrieve the stats for.

Note the player must be part of the 5000 most experienced players for its stats history to be recorded.
